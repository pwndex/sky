# Sky Practice

Прозрачный оверлей поверх всех окон для разучивания песен Sky: Music
(специально под формат `.skysheet` / `.json` с сайта specy.github.io/skyMusic).

Загружаешь свои песни → выбираешь из списка → приложение подсвечивает
клавиши инструмента (сетка Sky 3×5), которые нужно нажать. Оно слушает твою
клавиатуру: пока не сыграешь текущую комбинацию — дальше не пойдёт. Следующая
комбинация всегда подсвечена другим цветом, чтобы быть готовым.

## Что умеет
- Прозрачное окно без рамки, всегда поверх других окон (кнопка 📌 — вкл/выкл).
- Инструмент Sky 3×5 (15 пэдов) с точными иконками как на сайте: ромб-со-стрелками
  (корневые C), простой ромб (D F G B), круг (E G A) — бирюзово-зелёные.
- Ноты фиксированы: `C D E F G / A B C D E / F G A B C`. Структура не меняется.
- Загрузка одного или нескольких `.skysheet` / `.json`, `select` со списком песен.
- Трекинг нажатий: аккорд засчитывается, только когда нажаты **все** его ноты.
- Подсветка: **бордовый/красный** — играть сейчас, **приглушённо-серый** — следующая комбинация.
- Кнопки Restart / Back / Skip, пробел = пропустить шаг. Окно тянется за верхнюю полосу.

## Сохранение и один экземпляр
- **Все настройки сохраняются между запусками**: биндинги клавиш, загруженные песни,
  выбранная песня, прозрачность фона, положение окна и режим «поверх всех».
  Хранится в системной папке настроек (`AppConfig/SkyPractice/settings.json`),
  кроссплатформенно. Восстанавливается при следующем открытии автоматически.
- **Только один экземпляр**: если аппка уже открыта и ты запускаешь её снова —
  вместо новой копии просто поднимается уже открытое окно. Никаких 100500 окон.
- Кнопка **Clear songs** — очистить список песен.

## Прозрачность
Ползунок **Background opacity** делает полупрозрачным только фон/панель окна —
**сами пэды (клавиши) всегда остаются непрозрачными и своих цветов**. Так удобно:
фон не мешает видеть игру под окном, а инструмент всегда чёткий.

## Поверх всех окон
Кнопка 📌 держит окно поверх остальных. Окно **не скрывается**, когда фокус
уходит на другое приложение (игру, браузер) — оно остаётся видимым всегда.

На macOS, чтобы оверлей плавал ещё и **над полноэкранными** приложениями и был
виден на всех рабочих столах (Spaces), установи опционально `pyobjc`:

```bash
pip install pyobjc-framework-Cocoa
```

Без него окно всё равно держится поверх обычных окон — просто полноэкранный
режим на маке живёт на отдельном Space, и туда оверлей без pyobjc не поднимается.
Для сборки `.app` с этой возможностью добавь `pyobjc-framework-Cocoa` перед
запуском PyInstaller.

## Перехват клавиш даже без фокуса
Приложение ловит нажатия **глобально** (через `pynput`) — работает, даже когда
фокус на игре. Поставь зависимости:

```bash
pip install -r requirements.txt
```

### macOS — обязательно выдать разрешения
Без них macOS **молча не отдаёт** нажатия приложению — это и есть причина, по
которой «не трекает без фокуса». Нужно включить приложение в двух местах:

1. **Системные настройки → Конфиденциальность и безопасность → Универсальный
   доступ (Accessibility)** — включить SkyPractice (или Терминал/Python, если
   запускаешь через `python3 sky_practice.py`).
2. **Системные настройки → Конфиденциальность и безопасность → Мониторинг ввода
   (Input Monitoring)** — включить там же.

При первом запуске приложение само попросит доступ (появится системное окно).
После включения **перезапусти приложение** — это важно, разрешение подхватывается
только при старте. В строке статуса приложение пишет, включён ли глобальный
захват: «Global key capture on» — значит работает.

Если разрешений нет — приложение не падает, а работает в запасном режиме: клавиши
ловятся, только когда его окно в фокусе, и в статусе висит подсказка, куда зайти.

### Почему нужен именно перезапуск
macOS проверяет доверие процесса один раз при старте слушателя. Выдал доступ —
закрой и открой заново.

## Биндинг клавиш
- **Кликни левой кнопкой** по пэду → нажми клавишу клавиатуры → она забиндится.
  На пэде вместо ноты теперь показывается твоя клавиша.
- Если клавиша уже была назначена другому пэду — старый пэд освобождается
  (никаких дублей).
- **Правый клик** по пэду (или Backspace/Delete в режиме назначения) — снять бинд;
  пэд снова показывает название ноты.
- **Reset keys** — вернуть стандартную раскладку `Y U I O P / H J K L ; / N M , . /`.
- Esc — отменить назначение.

## Раскладка по умолчанию (стандарт Sky)
```
Y U I O P
H J K L ;
N M , . /
```

## Запуск из исходника
```bash
pip install -r requirements.txt
python sky_practice.py
```

## Сборка (Windows / macOS)
```bash
pip install -r requirements.txt pyinstaller
pyinstaller SkyPractice.spec
```
- **macOS** → `dist/SkyPractice.app`
- **Windows** → папка `dist/SkyPractice/` c `SkyPractice.exe` внутри
  (распространяй **всю папку**, заархивируй её — `.exe` требует файлы рядом)
- **Linux** → `dist/SkyPractice/SkyPractice`

Сборка идёт в режиме **onedir** (папка, а не один файл) — это **намного быстрее
запускается**, потому что при каждом старте ничего не распаковывается во временную
папку. Плюс из сборки исключены неиспользуемые модули Qt — меньше размер, быстрее
холодный старт.

> Сборку нужно делать **на той ОС, под которую собираешь** — PyInstaller не
> кросс-компилирует. `.exe` собирается на Windows, `.app` — на Mac. Код общий.

## Заметки по трекингу клавиш
Приложение читает нажатия, когда его окно в фокусе (стандартный, безопасный
режим — не требует прав администратора). Если нужен **глобальный** перехват
клавиш (играть, когда в фокусе другое окно/игра), это делается библиотекой
`pynput` — могу добавить отдельным режимом, скажи. На macOS для глобального
режима система запросит разрешение в *Настройки → Конфиденциальность →
Мониторинг ввода*.

## Поддерживаемые форматы песен
- `recorded`: `notes = [[idx, time_ms, layer], ...]`
- `songNotes`: `[{ "time": ms, "key": "1Key9" }, ...]`
- `composed`: `columns` из композитора

Ноты в окне 60 мс группируются в один аккорд.
